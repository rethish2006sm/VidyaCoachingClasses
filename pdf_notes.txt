🔥 1. Overall Architecture (Big Picture)  
Frontend  
• React + Tailwind CSS   
• SPA (Single Page Application)  
• Fast navigation using React Router  
Backend  
• Node.js + Express (pure JS as you want)   
• MongoDB Atlas (3 databases)  
Databases  
1. Main DB   
o Courses, offers, results, leaderboard  
2. User DB   
o Students + Faculty  
3. Gallery DB   
o Images only (important for performance)  
👉 Why 3 DB?  
• Separation = faster queries + scalable   
• Gallery images are heavy → separate DB avoids slowing main queries  
 
🔥 2. Data Flow (VERY IMPO RTANT)  
Flow Example (Course Page)  
1. User opens CoursePage  
2. React calls API → /courses   
3. Backend fetches from MongoDB  
4. Data → sent as JSON  
5. React stores in state   
6. Components render UI  
👉 Use: 
• useEffect()  → fetch data  
• useState()  → store data  
• axios  or fetch

⚡ 4. PERFORMANCE (VERY 
IMPORTANT)  
 
🔥 1. Lazy Loading  
const Gallery = React.lazy(() => import("./Gallery" )); 
👉 Loads only when needed  
 
🔥 2. Image Optimization  
• Use .webp   
• Compress images  
• Use CDN (Cloudinary)  
 
🔥 3. Code Splitting  
• Split pages  
• Reduce bundle size  
 
🔥 4. Memoization  
React.memo(Component ) 
useMemo() 
useCallback () 
👉 Prevent re -render  
 
🔥 5. API Optimization  
• Use pagination  
• Fetch only required data  
 
🔥 6. Database Optimization  
• Index fields:  
o branch   
o standard   
o year   
 
🔥 7. Caching  
• Cache topper, leaderboard  
 
🔥 8. Avoid Heavy Libraries  
• Use lightweight sliders  
• Avoid unnecessary packages  
 
🔥 9. Fast Routing  
• Use React Router  
• Avoid full reload  
 
🔥 10. Minimize State Updates  
• Avoid multiple setState
